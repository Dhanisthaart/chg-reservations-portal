Data Entry Standards

Data Entry Standards – Individual Guests

Guest profiles comprise of basic non-varying information such as guest name, address, contact details, language, as well as statistical information about stays in the current and previous years. A profile also includes a listing of future reservations, previous stays and guest preferences.

Standards

  • Individual profile is a profile for only one person. In the case of husband and wife, or guest travelling with family, each should have their own individual profiles created separately.
  • All guest profiles should contain the absolute minimum fields below. The 4 fields below are the main ones which guarantee proper profile interface between systems:
    Last Name | First Name | Country of Residence | Email address
  • The other fields listed below should also be correctly entered when available:
    Address | City, State / Province | Telephone numbers | Company | Preferences | Notes (Central Comment) | VIP Status

Considerations in Character Encoding

  • All data are to be entered in proper (upper, lower) case where applicable using the 26 character English (UK) alphabet. For special characters of a local language (i.e. ö,ë), they need to be adapted to the International spelling. However, in Opera central environment, special local characters are supported in certain fields like Last Name, First Name or Address Fields (Street, Building and City).
  • Data on the profile main screen should always be entered in English language. Any other local languages like German, French, Chinese, Japanese, etc. should be entered into the equivalent English characters.
  • Commas and period and the following symbols are not to be used in Name or Address elements:
  • ( ) Parentheses
  • " " Double quotes
  • = Double hyphen
  • [ ] Brackets
  • ... Ellipsis
  • : Colon
  • ; Semicolon
  • # Pound sign
  • % Percent
  • ^ Carat
  • * Asterisks
  • Punctuation Exceptions: In the case of a private registered guest, proceeding the guest's last name (PR) is allowed (e.g. (PR) Cruise). Use @ only in conjunction with an e-mail address (e.g. john@raffles.com).

Guest Name

FIELDRULES
Title
  • Enter the guest's title into the title field
  • The title is courtesy prefixed to the name of a person. It is usually written in the abbreviated form i.e. Mr., Mrs., etc. The title changes according to the guest's language.
  • In GHA OCIS, only 3 titles will be configured – Mr., Mrs., Ms.
Last Name
  • Enter the guest's full last name and in the format provided by the guest or in reference to the passport, ID card or driver's license entry.
  • Use a hyphen to separate a compound name (e.g. Jones-Smith) if one is provided
  • Two worded last name should appear with a space between names (e.g. La Salle)
  • Use apostrophes (') when the symbol is part of the proper name. Omit spaces before and after names with apostrophes (e.g. O'Brien)
  • All name beginning with Mc or Mac will be in the form of MacDonald, McKinley (no spaces between)
  • All last names beginning with Al, the Al will be placed as it is. Al Hussein should be written Al Hussein.
First Names
  • Enter the guest's full first name in the First Name field.
  • If the guest has two or more first names enter them into the first name field only (e.g. Mary Katherine / Clive Stafford Andrew).
  • If the guest has an English and Chinese (or any local/ethnic dialect) first names, enter the English name first followed by the Chinese / local / ethnic name in the First Name field. (e.g. Jacky Hok Yau)
  • Do not use the short forms of the first name (e.g. Wm for William)
  • The first name field should only have one person's name. In the case of husband and wife, each should have their own individual profiles created separately. For example: Mr & Mrs James & Lily Potter, the first name field in the husband profile should only carry 'James' but never be 'James & Lily'.
  • All name with bin or binte or binti, will be written without it, Jamal bin Al Hussein, will be written as Last name: Al Hussein and First name: Jamal.
Middle NameDo not use middle name field. First Name and Middle Name must be entered into First Name field.
SuffixesAll suffixes such as Junior, Senior, II, III will be entered into the last name field (no periods) and spelled out completely – do not abbreviate.

The last and first name fields are used differently in different countries. In a situation where you are dealing with a guest name of a nation that you are not familiar with, please verify with the guest how the name should be entered.

Address

  • Type of Address: Specify the type of address by selecting from the dropdown list if applicable. Examples: Home, Business
  • Address with 1 line (Street Number + Directional (if applicable) + Street Name)
    Address 1 is always mandatory and must never be empty. If there is one line of address, enter it into Address Line 1.
    Examples: 123 N Rose, 212 5th Ave
  • Address with 2 lines (Line 1: Street Number + Directional + Street name, Line 2: Building Name + Unit number)
    If there are two lines of address, enter the primary address into Address Line 1 and the unit number into Address Line 2.
    Examples: 16 Willow Dr Apt 205, 23 Hampton Pl, Icon Tower Ste 1210
  • Address entry rules:
    Numbers are to be written in the numeric form e.g. 2, and not "two". All addresses will be entered in mixed upper and lower case – no periods used for abbreviations. Do not create abbreviations for the fully spelt name.
  • Postal / Zip Code:
    This field must always be entered. If a US zip code has a zip+4, separate the first five digits from the last four digits with a hyphen.
    Examples: 039192, 85255-1234
  • City:
    This is used for the City / Suburb / District / Town / Village. In some countries, e.g. Australia, the name of the suburb is more important than the city for mailing purposes. Hence, the name of the suburb should be entered into this field. Do not abbreviate city names.
    Examples: Paddington (not Sydney), Los Angeles (not LA)
  • Country: Choose the correct country code from the list provided in Opera.

Business / Company

RULEEXAMPLE
Company Suffix: suffixes such as Co. Inc, Corp are not considered part of the company name and should not be used unless it is part of the actual company name – if used, do not abbreviateTexaco Inc. → Texaco, Boston Edison Co → Boston Edison, Bain and Company, Boston Company
Do not use the ampersand (&) symbol, type out conjunctions in fullProctor and Gamble, Johnson and Johnson
If a company name has more than two names, type without punctuationSears Roebuck and Company, Foley Hoag and Eliot
If a company name contains initials, type without punctuation and without spacesIBM, EF Hutton
Unless a number in a company name is formally numeric, it should be spelled outFirst National Bank, Saks Fifth Avenue

Phone Numbers

  • All telephone, fax and mobile numbers must have the country and area code regardless of local or International number. Example: +12121234567
  • Do not leave any spaces between the country code, area code and the number block. There should not be any "( )" for the area code and spaces or hyphen -, / for the telephone number block. This is to ensure consistency even if the mobile number is not from your country and you are not able to identify the prefix number.

    Examples:
    • +842412345678 (Vietnam)
    • +6621234567 (Thailand)
    • +6512345678 (Singapore)
    • +623612091888 (Indonesia)
    • +8613910998888 (China)
    • +12121234567 (USA)
  • Some countries have a leading zero in front of the area code. It is required to include this zero when dialling the number.

    Example:
    • +390277037734 (Milan, Italy)
  • NOTE: A minimum of 7 digits is required in the string for all telephone, fax and mobile phone numbers for interface with GHA and Cendyn. Numbers with less than 7 digits will be rejected during upload and then deleted locally as a result.
  • INCORRECT!

    Examples:
    • 1 (212) 1234567
    • 0049 (30) 2261-0
    • 97156

Email

  • A typical e-mail address has no spacing and is entirely in lower case. There is sometimes a combination of letters and numbers.
  • Examples:
    • franksmith@company.com
    • jacky1971@gmx.net
    • mike_fandango@yahoo.com
    • mike.fandango@gha.com

Membership Numbers

  • DISCOVERY membership numbers are numeric, 10 digits in length and are entered without blanks. A DISCOVERY membership number can only be assigned to one individual profile.

    Example:
    • 8671983522
  • LHW Leaders Club membership numbers are from 5 to 7 digits, no spaces, all numbers.
  • INCORRECT! blank spaces, too many digits.

    Example:
    • 1362 9036 874
